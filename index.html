AxisLink

Lead Developer

Core Concept

An intelligent, self-aligning antenna system that acts as a "dynamic signal spotlight." It is designed to solve the "last 50 meters" problem of unreliable wireless connections by actively finding a device and physically pointing a focused beam of data directly at it.

Core Technologies

  • Sensing: Wi-Fi RSSI (with a planned upgrade to high-precision Wi-Fi CSI).
  • Actuation: SG90 servo motors for mechanical beam steering.
  • Communication: Decentralized, low-latency ESP-NOW peer-to-peer protocol.
  • Microcontrollers: ESP32-WROOM-32U with U.FL connectors for external antennas.

Phased Development Roadmap

Phase 1 (Basic Tracker): A single-node proof-of-concept that scans 180° to find a device's general direction using RSSI.

Phase 2 (High-Performance Tracker): Upgrades the single node with a high-gain directional antenna to achieve longer range and a more reliable link.

Phase 3 (Coordinated Swarm): A 3-node master-slave system that performs parallel "sector scanning" for rapid target acquisition and signal optimization.

Phase 4 (Expert System): The final goal is to integrate Wi-Fi CSI to create a hybrid, high-precision tracking algorithm (RSSI for coarse lock, CSI for fine lock).

Software & Prototyping

The project's firmware is being developed professionally using PlatformIO within Visual Studio Code. The code is organized into two distinct projects, `AxisLink-Brain` and `AxisLink-Observer`, allowing for robust and scalable development.

Potential Applications

  • Creating low-cost, point-to-point "virtual cables."
  • Enhancing 4G/5G signals by locking onto distant cell towers.
  • Smart tracking for robotics, drones, and security systems.

Real-Time Temperature Monitoring System

Foundational Project

Objective

To design and build a simple embedded system capable of measuring ambient temperature and displaying the reading in Celsius on a character LCD. This project served as a practical introduction to sensor integration and data output using a microcontroller.

Components Used

  • Arduino UNO R3 Microcontroller
  • LM35 Analog Temperature Sensor
  • 16x2 Character LCD Display
  • 10kΩ Potentiometer (for LCD contrast)
  • Breadboard and Jumper Wires

Methodology

The LM35 sensor was interfaced with an analog input pin on the Arduino. The sensor produces a voltage output that is linearly proportional to the temperature. The Arduino's analog-to-digital converter (ADC) was used to read this voltage and convert it into a temperature value using a simple formula in the code. The `LiquidCrystal.h` library was utilized to send the formatted temperature data to the 16x2 LCD for display.

Outcome

A functional prototype that accurately displays the room temperature, updating every two seconds. The project successfully demonstrated the core principles of interfacing analog sensors with a microcontroller.

Sound-Activated Alarm Trigger

Foundational Project

Objective

To create a basic sound-sensitive circuit that activates a buzzer and an LED when a noise level exceeds a predefined threshold. This project explored the use of digital sensors and conditional logic to trigger an output.

Components Used

  • Arduino UNO R3 Microcontroller
  • KY-038 Sound Detection Sensor Module
  • 5V Active Buzzer
  • LED (for visual indication)
  • Breadboard and Jumper Wires

Methodology

The digital output pin of the sound sensor was connected to a digital input on the Arduino. The sensor's sensitivity was calibrated using its onboard potentiometer. The Arduino was programmed to continuously monitor the sensor's pin state. When a sound exceeding the threshold is detected, the sensor's output changes state. An `if` statement in the code detects this change and sends a signal to activate the buzzer and LED for a short duration.

Outcome

A reliable system that triggers a visual and audible alarm in response to a distinct sound, serving as a simple proof-of-concept for sound-activated control systems.
